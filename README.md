## 🤝 How to Contribute

We love contributors! Whether you are a beginner or a veteran, your input is valuable.

### 1️⃣ Setup Your Environment
1.  **Fork** the repository to your own account.
2.  **Clone** your fork locally:
    ```bash
    git clone https://github.com/your-username/nextgen.git
    cd nextgen
    ```
3.  **Install** dependencies:
    ```bash
    npm install  # or yarn install
    ```

### 2️⃣ The Workflow
* **Branch:** Create a branch for your work: `git checkout -b feature/your-feature-name`.
* **Develop:** Run the project locally using `npm run dev`.
* **Commit:** Use descriptive messages: `git commit -m "feat: add responsive navbar"`.
* **Push:** Send changes to your fork: `git push origin feature/your-feature-name`.

### 3️⃣ Submit & Review
Open a **Pull Request (PR)** from your fork. Provide a brief summary of your changes and include screenshots for UI updates.

---

## 📋 Contribution Standards
* **Clean Code:** Follow the existing linting and style rules.
* **Atomic PRs:** Keep your pull requests focused on a single feature or fix.
* **Documentation:** Update the `docs/` folder or comments if you change logic.
* **Respect:** Maintain a collaborative and supportive tone in PR reviews.
